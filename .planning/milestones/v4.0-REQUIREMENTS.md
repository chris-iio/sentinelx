# Requirements: v4.0 Universal Threat Intel Hub

**Defined:** 2026-03-02
**Core Value:** Expand from 3 hardcoded providers to 8+ via plugin-style registry architecture with unified results UX.
**Scope:** Provider registry refactor, 5 new threat intel providers, results UX upgrade.
**Foundation:** Provider Protocol (typing.Protocol), ProviderRegistry, ConfigStore multi-provider.

## v4.0 Requirements

### Registry Architecture

- [x] **REG-01**: Provider protocol with name, supported_types, requires_api_key, lookup(), is_configured()
- [x] **REG-02**: ProviderRegistry manages adapter registration and lookup by IOC type
- [x] **REG-03**: Orchestrator queries registry instead of hardcoding adapter lists
- [x] **REG-04**: ConfigStore supports multi-provider API key storage via [providers] INI section
- [x] **REG-05**: Settings page dynamically renders provider cards from registered providers

### Shodan InternetDB

- [x] **SHOD-01**: Shodan InternetDB enriches IP addresses without requiring an API key
- [x] **SHOD-02**: Adapter added via one file + one registration line — no orchestrator changes

### Free-Key Providers

- [x] **URL-01**: URLhaus adapter enriches URL, IP, domain, and hash IOCs
- [x] **OTX-01**: OTX AlienVault adapter enriches all 8 IOC types including CVE
- [x] **GREY-01**: GreyNoise Community adapter enriches IP addresses with riot/noise classification
- [x] **ABUSE-01**: AbuseIPDB adapter enriches IP addresses with confidence score verdicts
- [x] **MULTI-01**: Each provider enriches its supported types when configured with an API key
- [x] **MULTI-02**: Unconfigured providers are gracefully skipped without errors

### Results UX

- [x] **UX-01**: Each IOC card shows unified verdict summary aggregated across all providers
- [x] **UX-02**: Clicking a card expands to show per-provider detail rows with individual results
- [x] **UX-03**: Provider status indicators show which providers contributed vs. skipped vs. errored
- [x] **UX-04**: Settings page shows all registered providers with configuration status
- [x] **UX-05**: E2E tests pass for the new results layout
